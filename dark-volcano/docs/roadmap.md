# Dark Volcano - Development Roadmap

## Phase 1: Foundation & Core Systems
**Goal**: Establish basic game framework and core mechanics

### Deliverables:
- Basic game engine setup and project structure
- 3x3 party composition grid interface with Circle/Box formations
- Simple unit placement and movement system
- Basic FFTA class system (3-5 starter classes)
- Circle formation persistent aura system
- Box formation center aura unit system
- Fundamental item gravity mechanics
- Initial Tron-inspired visual style implementation

### Success Criteria:
- Player can place units in Circle (7 units) or Box (up to 9 units) formations
- Circle formations provide persistent aura to all 7 units
- Box formations with center aura unit affect 5 specific positions
- Aura scaling based on number of affected units works correctly
- Units have distinct visual styles and basic stats
- Items fall when not held by units
- Basic laser weapon effects functional

## Phase 2: Combat & Weapons
**Goal**: Implement the unique combat system with ricochet mechanics

### Deliverables:
- Laser weapon system with explosion/ricochet mechanics
- Combat calculation engine
- Visual effects for weapon impacts and reformation
- Basic AI for enemy units
- Simple battle resolution system

### Success Criteria:
- Weapons explode and reform as described in vision
- Combat feels satisfying and visually impressive
- Basic party vs party combat functional
- Performance acceptable for multiple simultaneous battles

## Phase 3: Strategic Map & Buildings
**Goal**: Create the grand strategy layer with building system

### Deliverables:
- Multi-scale zoomable map system
- Terrain-based building placement
- Basic resource production and flow
- Location conquest mechanics
- Simple economic simulation

### Success Criteria:
- Players can zoom between tactical and strategic views
- Buildings produce resources based on terrain
- Resource distribution between locations works
- Map feels alive and strategic

## Phase 4: Economy & Equipment
**Goal**: Implement the complex equipment and economy systems

### Deliverables:
- Hearts of Iron 4 style unit designer
- Equipment production and distribution
- Magic item identification system
  -- magic items claimed from battlefield loot must be taken to base before use
- Battlefield looting mechanics
  -- mundane items are turned into scrap which can be repurposed or sold
- Advanced resource management
  -- items are comprised of other items and labor. workshops unlock connections.

### Success Criteria:
- Unit customization is deep but accessible
- Economy feels balanced and engaging
- Item management creates meaningful decisions
- Equipment visually affects unit appearance

## Phase 5: Advanced Features & Polish
**Goal**: Add remaining features and polish for release

### Deliverables:
- Complete FFTA class system
- Advanced building types and terrain interactions
- Multiplayer support
- Audio system with "electric submarine" theme
- Tutorial and progression systems
- Performance optimization

### Success Criteria:
- All vision features implemented
- Game is fun and polished
- Performance is solid across target platforms
- Ready for beta testing

## Phase 6: Release & Post-Launch
**Goal**: Launch and iterate based on player feedback

### Deliverables:
- Public release
- Community feedback integration
- Additional content and features
- Ongoing balance updates

### Success Criteria:
- Successful launch
- Positive community reception
- Stable player base
- Foundation for future content
